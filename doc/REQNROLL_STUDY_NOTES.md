# Reqnroll Study Notes

Reqnroll is the BDD tool used by this project.

BDD means Behavior Driven Development.

Simple idea:

```text
Write test scenarios in plain English.
Connect each English sentence to C# code.
Run those scenarios as automated tests.
```

Bangla:

```text
Reqnroll দিয়ে English-এর মতো test case লেখা যায়।
তারপর সেই English step গুলো C# method-এর সাথে connect করা হয়।
```

## Feature Files

Feature files use `.feature`.

This project has:

```text
SauceDemo.Tests/Features/Purchase.feature
```

A feature file is like a human-readable test plan.

Example:

```gherkin
Feature: SauceDemo purchase journey
```

Meaning:

```text
This file is about testing the SauceDemo purchase journey.
```

## Scenarios

A scenario is one test case.

Example:

```gherkin
Scenario: Happy path - user completes an end-to-end purchase
```

Meaning:

```text
This test checks that a normal user can buy a product successfully.
```

## Gherkin Keywords

### `Given`

Sets up the starting situation.

Example:

```gherkin
Given I am on the SauceDemo login page
```

Meaning:

```text
Start on the login page.
```

### `When`

Performs an action.

Example:

```gherkin
When I log in as a standard user
```

Meaning:

```text
Do the login action.
```

### `Then`

Checks the result.

Example:

```gherkin
Then I should see the order confirmation message
```

Meaning:

```text
Check that the order success message is visible.
```

### `And`

Continues the previous type of step.

Example:

```gherkin
When I log in as a standard user
And I add the "Sauce Labs Backpack" product to the cart
```

Meaning:

```text
Both lines are actions.
```

Bangla:

```text
Given = test কোথা থেকে শুরু হবে
When = user কী action করবে
Then = result কী হওয়া উচিত
And = আগের step type চালিয়ে যাওয়া
```

## Step Definitions

Step definitions connect Gherkin text to C# code.

This project has:

```text
SauceDemo.Tests/StepDefinitions/PurchaseStepDefinitions.cs
```

Example:

```csharp
[When("I log in as a standard user")]
public void WhenILogInAsAStandardUser()
{
    LoginPage.Login(StandardUsername, StandardPassword);
}
```

ELI5 meaning:

```text
When Reqnroll sees this sentence:
"I log in as a standard user"

Run this C# method.
```

## `[Binding]`

Example:

```csharp
[Binding]
public class PurchaseStepDefinitions
```

Meaning:

```text
Reqnroll, this class contains step definitions.
Please look inside it.
```

Without `[Binding]`, Reqnroll will not know the class has step code.

## Step Parameters

Gherkin can pass values into C#.

Example from the feature file:

```gherkin
When I log in with username "wrong_user" and password "wrong_password"
```

Matching C#:

```csharp
[When("I log in with username {string} and password {string}")]
public void WhenILogInWithUsernameAndPassword(string username, string password)
{
    LoginPage.Login(username, password);
}
```

Meaning:

```text
"wrong_user" goes into username.
"wrong_password" goes into password.
```

Bangla:

```text
Gherkin-এর quote-এর ভিতরের value C# method parameter-এ যায়।
যেমন "wrong_user" username variable-এ যায়।
```

## Hooks

Hooks run code before or after scenarios.

This project has:

```text
SauceDemo.Tests/Hooks/DriverHooks.cs
```

### `[BeforeScenario]`

Runs before each scenario.

In this project, it opens Chrome.

```csharp
[BeforeScenario]
public void OpenBrowser()
```

Meaning:

```text
Before every scenario, open a fresh browser.
```

### `[AfterScenario]`

Runs after each scenario.

In this project, it closes Chrome.

```csharp
[AfterScenario]
public void CloseBrowser()
```

Meaning:

```text
After every scenario, close the browser.
```

## Generated `.feature.cs` Files

Reqnroll may create files like:

```text
Purchase.feature.cs
```

Do not edit this file.

Simple idea:

```text
You write Purchase.feature.
Reqnroll generates Purchase.feature.cs.
NUnit runs the generated test code.
```

This project ignores generated feature code in `.gitignore`:

```gitignore
*.feature.cs
```

## Reqnroll Flow

When you run:

```bash
dotnet test SauceDemoAutomation.slnx
```

This happens:

```text
1. Reqnroll reads Purchase.feature.
2. Reqnroll finds matching step definitions.
3. BeforeScenario opens Chrome.
4. Gherkin steps run one by one.
5. Then steps use NUnit assertions.
6. AfterScenario closes Chrome.
7. NUnit reports pass or fail.
```

## Reqnroll vs Page Objects

Reqnroll should describe behavior.

Page objects should control web pages.

Good:

```text
Step definition says:
LoginPage.Login(username, password)
```

Not good:

```text
Step definition says:
driver.FindElement(By.Id("user-name")).SendKeys(username)
```

Why?

```text
The step definition should stay readable.
The page object should hide Selenium details.
```

## What You Should Remember

- Reqnroll lets you write tests in Gherkin.
- `.feature` files contain scenarios.
- Step definitions connect Gherkin to C#.
- Hooks open and close the browser.
- Reqnroll works with NUnit in this project.
- Do not edit generated `.feature.cs` files.

## Reqnroll vs Playwright Test

Playwright usually keeps the test and code together:

```typescript
test("standard user can login", async ({ page }) => {
  await page.goto("https://www.saucedemo.com");
  await loginPage.login("standard_user", "secret_sauce");
  await expect(page).toHaveURL(/inventory/);
});
```

Reqnroll separates the English scenario from the C# code.

Gherkin:

```gherkin
When I log in as a standard user
```

C# step definition:

```csharp
[When("I log in as a standard user")]
public void WhenILogInAsAStandardUser()
{
    LoginPage.Login(StandardUsername, StandardPassword);
}
```

Quick map:

```text
Playwright test(...)          ~= Reqnroll Scenario
Playwright test.step(...)     ~= Gherkin Given/When/Then step
Playwright beforeEach         ~= Reqnroll BeforeScenario
Playwright afterEach          ~= Reqnroll AfterScenario
Playwright expect             ~= NUnit Assert.That
Playwright Page object        ~= Selenium Page Object class
```

Main difference:

```text
Playwright tests are code-first.
Reqnroll tests are English-first, then mapped to code.
```

Bangla:

```text
Playwright-এ সাধারণত test সরাসরি code হিসেবে লেখা হয়।
Reqnroll-এ আগে English scenario লেখা হয়, তারপর C# step definition লেখা হয়।
```

## Bangla Quick Reference

```text
Reqnroll = C#/.NET-এর BDD tool
BDD = behavior English-এর মতো লিখে test করা
Feature = বড় topic বা feature
Scenario = একটা test case
Given = starting condition
When = action
Then = expected result
And = আগের step type continue করা
Step Definition = Gherkin line-এর C# code
Binding = Reqnroll-কে বলে এই class-এ step আছে
BeforeScenario = scenario-এর আগে run হয়
AfterScenario = scenario-এর পরে run হয়
```

Playwright-এর সাথে মিল:

```text
Scenario                 ~= test(...)
Given/When/Then          ~= test steps
Step Definition method   ~= test code function
BeforeScenario           ~= beforeEach
AfterScenario            ~= afterEach
NUnit Assert.That        ~= Playwright expect
```

সবচেয়ে গুরুত্বপূর্ণ:

```text
Reqnroll English-first।
Playwright সাধারণত code-first।
```
