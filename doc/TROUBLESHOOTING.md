## `dotnet test` Fails

Command:

```bash
dotnet test SauceDemoAutomation.slnx
```

What this command does:

```text
1. Reads the solution file.
2. Restores/builds projects if needed.
3. Runs NUnit tests generated by Reqnroll.
```

Bangla:

```text
dotnet test শুধু test চালায় না।
এটা আগে project build/restore করতেও পারে।
```

Check .NET installation:

```bash
dotnet --info
```

If this command fails, .NET SDK is not installed correctly.

Bangla:

```text
dotnet --info fail করলে আগে .NET SDK install/check করতে হবে।
```

First check:

```text
1. Error message কোন file দেখাচ্ছে?
2. Error compile-time নাকি test-time?
3. Browser open/element find/assertion - কোন জায়গায় fail করছে?
```

Bangla:

```text
প্রথমে error-এর line number দেখুন।
তারপর বুঝুন fail build করার সময়, নাকি browser test চালানোর সময়।
```

## Build Error

Example:

```text
The type or namespace name ... could not be found
```

Meaning:

```text
C# কোনো class/library খুঁজে পাচ্ছে না।
```

.NET files to check:

```text
SauceDemoAutomation.slnx
SauceDemo.Framework/SauceDemo.Framework.csproj
SauceDemo.Tests/SauceDemo.Tests.csproj
```

Possible fixes:

```text
Check using statement.
Check namespace.
Check project reference.
Run dotnet restore.
```

Bangla:

```text
Build error মানে code compile হচ্ছে না।
সাধারণত using, namespace, package, বা project reference issue হয়।
```

Playwright comparison:

```text
C# namespace/import error ~= JS/TS import/module not found
```

## ChromeDriver Error

Possible error:

```text
ChromeDriver only supports Chrome version...
```

Meaning:

```text
Chrome browser version আর ChromeDriver version match করছে না।
```

Possible fixes:

```text
Update Selenium.WebDriver.ChromeDriver package.
Update Chrome browser.
Run dotnet restore.
```

Bangla:

```text
Chrome আর ChromeDriver version mismatch হলে browser start নাও হতে পারে।
```

## Element Not Found

Possible error:

```text
NoSuchElementException
```

Meaning:

```text
Selenium page-এর element খুঁজে পাচ্ছে না।
```

Possible reasons:

```text
Wrong locator.
Page not loaded yet.
You are on the wrong page.
Element text/id changed.
```

Possible fixes:

```text
Check the locator in the page class.
Add or improve WebDriverWait.
Check previous step really navigated to the correct page.
```

Playwright comparison:

```text
Selenium NoSuchElementException ~= Playwright locator timeout
```

Bangla:

```text
element না পেলে আগে locator check করুন।
তারপর দেখুন page load হওয়ার আগে code run করছে কিনা।
```

## Element Not Interactable

Possible error:

```text
ElementNotInteractableException
```

Meaning:

```text
Element আছে, কিন্তু এখন click/type করা যাচ্ছে না।
```

Possible reasons:

```text
Element hidden.
Page transition still happening.
Element is outside visible screen.
Another element is covering it.
```

Possible fixes:

```text
Wait until visible and enabled.
Scroll element into view.
Click after page transition finishes.
```

In this project, see:

```text
SauceDemo.Framework/Pages/CheckoutPage.cs
```

Bangla:

```text
element পাওয়া গেছে মানেই click করা যাবে এমন না।
visible/enabled হওয়া পর্যন্ত wait করতে হয়।
```

## Assertion Fails

Possible error:

```text
Expected: ...
But was: ...
```

Meaning:

```text
Test ran, but actual result expected result-এর সাথে মেলেনি।
```

Possible fixes:

```text
Check expected text.
Check app actually reached correct page.
Check whether assertion is too strict.
Wait for final state before assertion.
```

Playwright comparison:

```text
NUnit assertion fail ~= Playwright expect fail
```

Bangla:

```text
Assertion fail মানে code চলেছে, কিন্তু result expected ছিল না।
```

## GitHub Actions Fails

Check:

```text
1. Did restore pass?
2. Did build pass?
3. Did tests start?
4. Which test failed?
5. Was it browser/Chrome related?
```

Common CI fixes:

```text
Run Chrome headless.
Use --no-sandbox.
Use --disable-dev-shm-usage.
Use fixed window size.
```

This project already uses those Chrome settings in:

```text
SauceDemo.Tests/Hooks/DriverHooks.cs
```

Playwright comparison:

```text
Playwright CI often needs npx playwright install --with-deps.
Selenium CI often needs Chrome/ChromeDriver/headless settings.
```

Bangla:

```text
local machine-এ pass করলেও CI-তে fail করতে পারে।
কারণ CI machine headless Linux environment।
```

## Clean Build Folders

If build output gets weird, you can delete generated folders:

```bash
rm -rf SauceDemo.Framework/bin SauceDemo.Framework/obj
rm -rf SauceDemo.Tests/bin SauceDemo.Tests/obj
dotnet test SauceDemoAutomation.slnx
```

Playwright comparison:

```text
Deleting bin/obj ~= deleting dist/test-results/node_modules cache-like generated files
```

Bangla:

```text
bin আর obj generated folder।
delete করলে problem নেই, dotnet আবার বানাবে।
```

## Best Debugging Order

```text
1. Read the exact error message.
2. Check the file and line number.
3. If locator failed, inspect the page class.
4. If assertion failed, inspect expected vs actual value.
5. Run only after saving changes.
6. Re-run dotnet test.
```

Bangla:

```text
guess না করে error message follow করুন।
line number থেকে শুরু করুন।
```
